Developing plug-ins for FuturixImager
 
Back in the year 2000, Futuris Imager was the first Windows image viewer with plug-in system that was open to the public - nowadays FuturixImager plug-in system is vastly expanded and more powerful then ever.
 
 
FuturixImager plug-in is a Windows DLL (currently 32-bit only) which exports one required function (FxImgQuery) and one or more optional functions (FxImgOpen, FxImgSave, etc). All plug-ins are located in FuturixImager folder and discovered by the program when user scans for new plug-ins in the "Options" dialog.
 
 

How to create a new plug-in:

  1. Create new empty native DLL project in your development environment (currently supported are Microsoft Visual C++ 2005 and Borland Delphi 7, but other environments can be used - as long as they support native 32-bit DLLs, DLL exports and callbacks).
  2. Write and export FxImgQuery() function - this is the only required function for FuturixImager plug-in system. Inside it you must pass appropriate data to the callback - FxImgQuery() will be called by FuturixImager during plug-in scan.
  3. Depending on what functionality you want to have in your plug-in, define required functions and export them.
  4. Test the result. And that's it :-)
 
 
Functions to export:
 
Function Description
FxImgQuery() Required function - without it DLL will not be recognized as FuturixImager plug-in.
  Called by FuturixImager during plug-in scan to get details of plug-in functionality.
  From FxImgQuery() call info_call with parameters describing your plug-in (e.g. loaded extensions, filter names, etc).
   
FxImgCfg() Used to display plug-in configuration dialog (if needed).
  In FxImgQuery() your plug-in should tell FuturixImager that it has configuration dialog (and provide name for it - you can have many dialogs with different names in the same DLL).
  Dialogs will be available to user in FuturixImager "Options" dialog.
   
FxImgOpen() Used to open plain images (not multi-page or animated).
  Any kind of GUI in this function is forbidden.
  Used in "Open" dialog and for thumbnail extraction.
  To enable: in FxImgQuery() call info_call with PS_FOPEN and file format extension (aka "bmp").
   
FxImgSave() Used to save simple images to specific filename (support for saving multi-page images planned for the future).
  Simple GUI for saving options is permitted, for more complex GUI use FxImgExport().
  Used in "Save As " dialog.
  To enable: in FxImgQuery() call info_call with PS_FSAVE and file format extension (aka "bmp").
   
FxImgImport() Used to get some bitmap out of nowhere (scanner, clipboard, etc) and view it in the program.
  Used in "File" -> "Import" menu.
   
FxImgExport() Used to save current bitmap to external devices/sources (print, send by e-mail, etc).
  Used in "File" -> "Export" menu.
  FuturixImager does not provide file name to save to, if you need it then use FxImgSave().
   
FxImgFilter() Used to transform current image (or current selection) and return it back to the program.
  Used in "Editor" dialog.
  Use preview_call to display preview to user.
   
FxImgTool() Used for everything that does not fit into other functions.
  Used in "Tools" menu.
   
FxImgInfo() Used to read some information about image and display it to the user as simple set of strings.
  Used in "Image Information" dialog.
   
FxImgAnimStart() Used to play animated or streaming images.
FxImgAnimRestart() This set of functions is deprecated and may be removed in one of the future versions of FuturixImager.
FxImgAnimGetFrame() You should also export FxImgOpen() and return first frame of the animation there (it will be used as thumbnail).
FxImgAnimStop() Used in "Open" dialog.
  All control over duration of animation delegated to plug-in, FuturixImager does not have internal frame counter - it just asks for the next frame to display and for the delay to display this frame for.
   
FxImgMultiStart() Used to view multi-page images.
FxImgMultiGetPage() User controls which page to display.
FxImgMultiStop() You should also export FxImgOpen() and return first page there (it will be used as thumbnail).
  Used in "Open" dialog.
 
 
Folder structure:
 
Folder Description
\bin\ Binaries for some plug-ins (for plug-ins written in C++) will be copied to this folder.
   
\source\plugins\ Plug-in sources (binaries are included with current version of FuturixImager).
\source\shared\ Several source files from FuturixImager (these files contain useful functions and constants - you cannot compile provided plug-ins without them). See explanation below.
   
\thirdparty\ Delphi VCL components that are used by provided plug-ins (you will need to install these components if you want to compile some plug-ins). Components in this folder are modified and not the original versions from developers' websites.
 
 
Headers explained:
 
Header Description
c_const.pas Header for constants and function definitions used by FuturixImager plug-ins (Delphi).
   
c_locales.pas Localization support functions (Delphi).
   
c_reg.pas Extended version of TRegistry (Delphi).
   
c_themes.pas Theming support functions (Delphi).
   
c_utils.pas Utility functions (Delphi).
   
c_wndpos.pas Functions to quickly save/restore window coordinates and/or position to registry (Delphi).
   
fximagerdefs.h Header for constants and function definitions used by FuturixImager plug-ins (C++).
   
h_chm.pas Headers with some Win32 API definitions converted to Pascal (Delphi).
h_nt.pas C++ versions are available in Microsoft's Platform SDK.
h_win.pas

Additional features:
 
Feature Description
File format descriptions It is possible to set nice description for specific file format, for example check FxImgQuery() function in fimg plug-in.
   
Rare file formats If you are implementing some rare file format or support for this format in your plug-in is incomplete - then it is better to hide this format from the list of file formats by default (this applies only to registration of this file format with FuturixImager - user will still be able to open it and see it in the "Open" dialog). For example on how to do this check FxImgQuery() function in fimg plug-in.
   
Plug-in roles It is possible for some plug-ins to get button on FuturixImager toolbar. Such plug-ins should export specific function and in FxImgQuery() function they should inform FuturixImager that they support specific role.
  Available roles are:
PR_SCAN - scanner functionality in FxImgImport()
PR_EMAIL - e-mail functionality in FxImgTool()
PR_CAPTURE - screen capture in FxImgImport()
PR_JPEGLL - JPEG lossless transformations in FxImgTool()
PR_HEX - HEX viewer/editor functionality in FxImgTool()
 
 

Tips:

  1. FuturixImager internally works only with 24-bit images, so all images that you pass to it (and get from it) should/will always be 24-bit!
     
  2. All exported FxImgQuery() functions should include valid call with PT_FNAME to give plug-in name to host application. The plug-in name should be in English.
     
  3. All functions in FuturixImager plug-in system should use CDECL calling convention without C++ function name mangling.
     
  4. Most functions in FuturixImager plug-in system have several common parameters:
    app (type HWND) - FuturixImager instance handle (use to load resources from fximager.exe if needed)
    wnd (type HWND) - handle of main FuturixImager window (use as parent for dialogs and message boxes)
    app_query (type APPCALLBACK) - internal FuturixImager callback (will be used in the future versions to get additional information from host application)
     
  5. Plug-ins created with this version of SDK will work only with FuturixImager 5.7 (and newer).
